# Title
Product of the Last K Numbers 



# Approach Name
Prefix Product with Zero Handling 



# Intuition
The task is to efficiently calculate the product of the last k numbers added. The naive approach of multiplying the last k elements directly would be slow for repeated queries. Instead, we can use a **prefix product** approach, where we maintain a running product of all elements.  
The main challenge arises when encountering a zero, which resets the product. To handle this, we:
- Clear the prefix product list whenever a zero is added.
- Start a new product sequence after the zero. 
 

# Approach
- **Using a Prefix Product List**:  
  Store cumulative products in an ArrayList<Integer> named products. The i-th element in this list represents the product of all numbers from the start up to the i-th number.
  *Reason*: Using cumulative products allows quick calculation of the product of any subarray by dividing the cumulative product at the end of the range by the cumulative product just before the start of the range.

- **Handling Zeros**:  
  - If 0 is added, clear the product list and start over by adding 1 as the base product.
  *Reason*: Any product including 0 becomes 0, so all prior products are invalidated.
  - If a non-zero number is added, multiply the last product by this number and append the result to the list.
  *Reason*: This maintains the running product sequence.

- **Querying the Product**:
  - To get the product of the last k numbers, we divide the most recent product by the product at the position k steps back.
  - If the requested range includes a zero (detected by insufficient entries in products), return 0.
  *Reason*: Zeros reset the product sequence, so if the range includes a zero, the product must be zero. 
 

# Logic of the code
1. **Initialization (ProductOfNumbers() method)**:  
   - Create an ArrayList<Integer> named products and initialize it with 1.  
     *Reason*: Initializing with 1 simplifies product calculations because multiplying by 1 has no effect.
   - Initialize prevProduct to 1.  
     *Reason*: This variable tracks the running product and is used to compute the next product efficiently.

2. **Adding a Number (add(int num) method)**:  
   - Check if num is 0.  
     - If true, clear the products list and add 1 to start a new sequence. Also, reset prevProduct to 1.  
       *Reason*: Since any product including 0 is 0, all prior products become invalid, so a new product sequence starts.
     - If false, multiply prevProduct by num and append it to the products list.  
       *Reason*: This maintains the cumulative product sequence.

3. **Getting the Product of the Last K Numbers (getProduct(int k) method)**:  
   - Calculate the index of the product k steps before the latest product (len - 1 - k).  
     *Reason*: This represents the start of the subarray whose product we want.
   - Check if this index is negative.  
     - If true, return 0.  
       *Reason*: A negative index means the requested range includes a zero, which resets the product.
     - If false, calculate the product as:
    
        int num1 = products.get(len - 1);
        int num2 = products.get(len - 1 - k);
       *Reason*: This uses the prefix product property:  
       ###### Product of last k numbers = Product up to last element / Product up to element just before last k

4. **Edge Cases Considered**:
   - Multiple consecutive zeros.
   - Querying for k greater than the number of elements since the last zero.
   - Querying after adding 0. 
 

# Complexity
- **Time Complexity**:  
  - add(int num):  
    $$O(1)$$ in most cases, as it only involves multiplication and appending to the list.  
    If num is 0, the list is cleared, which takes $$O(1)$$ amortized over multiple operations.
  - getProduct(int k):  
    $$O(1)$$, as it involves only array access and division.

- **Space Complexity**:  
  $$O(n)$$, where n is the number of elements added since the last zero. This is because the products list grows linearly with the number of additions.

class ProductOfNumbers {
    ArrayList<Integer> products;
    int prevProduct;

    public ProductOfNumbers() {
        this.products = new ArrayList<>();
        this.prevProduct = 1;
        products.add(1);
    }
    
    public void add(int num) {
        if(num == 0){
            products.clear();
            products.add(1);
            prevProduct = 1;
        }else{
            prevProduct *= num;
            products.add(prevProduct);
        }

    }
    
    public int getProduct(int k) {
        int len = products.size();
        if(len - 1 - k < 0){
            return 0;
        }
        int num1 = products.get(len - 1);
        int num2 = products.get(len - 1 - k);
        return num1 / num2;
    }
}
