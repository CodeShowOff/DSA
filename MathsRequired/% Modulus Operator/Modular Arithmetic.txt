Modular Arithmetic:

Properties:
we know that "%" gives the reminder i.e. 15 % 2 = 1 , here 1 is reminder left after dividing 15 by 2.

here; 15 -> Dividend , n
       2 -> Divisor , m 
       7 -> Quotient , m/n
       1 -> Remainder , r 

       we know : Dividend = Divisor * Quotient + Remainder
            so : n = m * (n/m) + r 
            =>   r = n - (n/m) * m

            lly, 15 = 2 * (15/2) + r
            =>   r = 15 - (15/2) * 2  => r = 1

the notation n mod m (for integers n and m ) to mean " n - (n/m) * m ".

Note: % is often called the "modulo operator" but in some instances, it's more correct to call it the "remainder operator"

Note: If -8 % 7 == 6, you're fine, but if it is -1, you'll need to adjust it by adding m to any negative results. 
      If you use Java/Kotlin, the standard library function Math.floorMod(n, m) does what we need.



"Basic" arithmetic : 
First off, some important identities about the modulo operator:

((a % m) + (b % m)) % m = (a + b) % m
((a % m) − (b % m)) % m = (a − b) % m
((a % m) * (b % m)) % m = (a * b) % m

Why Modulo 10^9 + 7?:
In competitive programming, problems often involve large integers.
To prevent integer overflows and handle large results efficiently, we use modulo arithmetic.
The value 10^9 + 7 (which is 1000000007) is commonly chosen as the modulo because:
It is a prime number (which ensures unique results).
It fits within the range of common integer data types (e.g., int).
Any prime number less than (2^{30}) would suffice to prevent overflows.

How It’s Used:
When you’re asked to “print the answer modulo 10^9 + 7,” it means you should compute the result and then take the modulo with this specific number.
For example, if you calculate (x^n), you would print ((x^n) \mod (10^9 + 7)).
